About Lou

The WaterLou Open Data Chatbot, or Lou for short, was created as a final project for GEOG 483. It is designed to allow users to interact with the City of Waterloo’s Open Data Portal in a friendlier way to discover new datasets, visualize them on a map, learn more about their metadata, and the fields they contain. Four key components make up the project:

Chat Interface: The chatbot interface is built with HTML/JavaScript and allows users to enter their questions to the bot and see responses. Leaflet powers the map functions and is extended via the Esri Leaflet library. Supporting CSS helps to style things appropriately.

Watson Assistant API: IBM’s Watson Assistant API allows us to perform natural language processing on input text to determine user intents and entities, as well develop conversational dialogue. We developed a custom build of this API to fit it to this project and allow us to trigger further actions when the server receives a response from Watson.

Waterloo Open Data Portal/ArcGIS REST: The Waterloo Open Data Portal is accessible via ArcGIS REST endpoints, which allowed us to build a dynamic and scalable application that contains minimal hardcoding and no local data. Spatial features and accompanying metadata are pulled live from the data portal through the REST interface.

Flask Server: Flask is a Python library that runs a custom server that handles processing in Python. Flask is the central point of this project and communicates to and from each of the other components quickly and effectively. The server handles requests to Watson and ArcGIS services and creates HTML response templates based on the returned data.
